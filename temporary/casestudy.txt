1. Synopsis 
SpendSmart is a full-stack Expense Tracker with Data Visualization that enables individuals to take 
complete control of their personal finances. Users can log daily expenses and income, organise 
transactions into custom categories, set per-category and overall monthly budgets, configure recurring 
transactions for automatic tracking, and gain deep financial insight through a rich dashboard of 
interactive charts and analytics. 
 
The platform supports two primary roles: Users (who manage their own financial data) and 
Administrators (who oversee the platform). Users can track both expenses and income across unlimited 
custom categories with colour and icon customisation, monitor budget limits with real-time progress 
bars and configurable alert thresholds, visualise their finances through pie charts (category 
breakdown), bar charts (income vs expense), line charts (daily trend, savings rate), and a yearly 
overview. Advanced analytics features include a spending forecast using historical trend analysis, a 
Financial Health Score computed from savings rate and budget adherence, and a cash flow summary. 
 
SpendSmart is architected as a microservices system modelled on the EShoppingZone reference 
pattern, with independent services for authentication/user management, expense tracking, income 
tracking, category management, budget management, analytics and reporting, recurring transaction 
automation, and notifications/alerts ‚Äî all integrated through a Spring MVC website controller layer.

4.5 Budget-Service 
The Budget-Service manages per-category and overall monthly budgets. Each Budget tracks 
spentAmount (updated atomically on every expense add/edit/delete via the Expense-Service) against a 
limitAmount, with a configurable alertThreshold percentage triggering a Notification-Service call when 
crossed. The getBudgetProgress operation returns a BudgetProgress DTO containing the percentage 
used, remaining amount, and alert status. A scheduled job resets budget periods at the start of each 
new period.

4.6 Analytics / Report-Service 
The Analytics/Report-Service is the data intelligence layer of the platform. It aggregates expense and 
income data from other services to produce the dashboard visualisation payloads. FinancialSnapshot

composite of savings rate (40%), budget adherence rate (40%), and expense-to-income ratio (20%). 
Spending forecasts use a 3-month trailing average with a momentum adjustment factor.

4.7 RecurringTransaction-Service 
The RecurringTransaction-Service manages automated transaction rules. Each rule stores a frequency, 
nextDueDate, and an optional endDate. A Spring @Scheduled job runs daily at midnight to find all 
active recurring transactions whose nextDueDate equals today, generates the corresponding expense 
or income entry via the Expense/Income-Service, and advances the nextDueDate by one frequency 
period. Users receive a 3-day advance reminder notification via the Notification-Service.

4.8 Notification / Alert-Service

5. Microservices Architecture Overview 
SpendSmart is structured as a set of independently deployable microservices following the 
EShoppingZone reference pattern. Each service owns its own database schema, REST API contract, 
service interface, and business logic. Synchronous inter-service calls use RestTemplate; asynchronous 
budget alerts, recurring transaction generation, and monthly summary emails are driven by RabbitMQ 
and Spring Scheduler.

6. Non-Functional Requirements 
Category 
Requirement 
Performance 
Dashboard chart data must load within 1.5 seconds; analytics queries 
are cached in Redis with a 10-minute TTL to reduce repeated 
aggregation cost 
Scalability 
Each microservice independently scalable via Docker/Kubernetes; 
Analytics-Service horizontally scaled for concurrent report generation 
Security 
Passwords stored as bcrypt hashes; JWT tokens expire in 24 hours; 
OAuth2 for Google login; all API endpoints are user-scoped ‚Äî 
cross-user data access is rejected at service layer 
Data Isolation 
Every repository query is parameterised with userId to ensure strict 
data isolation; no query can return data from a different user's account 
Budget Atomicity 
spentAmount updates on Budget are executed with database-level 
atomic increment/decrement to prevent inconsistency under 
concurrent expense additions 
Recurring Reliability 
Scheduled recurring job uses idempotency keys to prevent duplicate 
transaction generation on scheduler restarts 
Forecast Accuracy 
Spending forecast uses a weighted 3-month trailing average with an 
exponential smoothing factor; re-computed on each analytics request 
Audit Trail 
All admin actions (user suspension, deletion) and budget alert 
dispatches are logged with actor, timestamp, and entity reference 
Export Quality 
PDF reports render charts server-side using JFreeChart or Apache 
ECharts headless; CSV exports stream large datasets without 
memory overflow 
Usability 
Responsive dashboard on desktop and mobile; all charts are

8. Glossary 
Term 
Definition 
Expense 
A financial outflow transaction logged by a user with amount, category, 
date, and payment method 
Income 
A financial inflow transaction logged by a user with amount, source, 
category, and date 
Category 
A user-defined or system-default grouping label for organising expenses 
and income into meaningful themes 
Budget 
A monetary spending limit defined for a category or overall, tracked against 
actual spending for a given period 
Budget Progress 
A real-time ratio of spentAmount to limitAmount for a budget, expressed as 
a percentage and displayed as a progress bar 
Alert Threshold 
A configurable percentage (e.g., 80%) at which a budget warning 
notification is automatically triggered 
Recurring Transaction 
An automated financial rule that creates a real expense or income entry on 
a defined schedule (daily/weekly/monthly/etc.) 
nextDueDate 
The next calendar date on which a recurring transaction will be 
automatically generated 
Savings Rate 
Monthly net savings divided by total income, expressed as a percentage: 
(Income ‚àí Expenses) / Income √ó 100 
Net Savings 
The difference between total income and total expenses for a given period 
Financial Health Score 
A composite 0‚Äì100 score computed from savings rate, budget adherence 
rate, and expense-to-income ratio 
Spending Forecast 
A predicted next-month total expense amount computed using a weighted 
3-month trailing average with trend adjustment 
Cash Flow 
The net difference between total inflows (income) and total outflows 
(expenses) over a time period 
FinancialSnapshot 
A point-in-time record of total income, expenses, net savings, and savings 
rate for a specific month, written monthly by the scheduler 
Payment Method 
The mechanism used to pay for an expense: Cash, Card, UPI, Bank 
Transfer, or Wallet 
Income Source 
The origin of an income entry: Salary, Freelance, Business, Investment, 
Gift, or Other
